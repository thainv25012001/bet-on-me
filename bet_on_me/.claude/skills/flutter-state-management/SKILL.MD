When explaining or improving state management, always include:

1. **Start with an analogy**  
   Compare state management to a central control system coordinating different parts of an application.

2. **Draw a data flow diagram**  
   Use ASCII diagrams to show how state moves through the app.

3. **Walk through the logic**  
   Explain how UI, state, and business logic interact.

4. **Provide improved implementation examples**  
   Suggest modern approaches such as Riverpod or Bloc.

5. **Highlight a gotcha**  
   Mention common mistakes like storing too much logic in UI widgets.

Focus on predictable, testable state management patterns.
